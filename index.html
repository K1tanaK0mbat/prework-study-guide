HTML

  • HTML: coding used to create the structure of webpages by adding elements that setup the framework for web pages.
  • Elements have opening tag (words) and closing tag (/words)
  • head element contains metadata that isnt visible on webdage but helps with rendering
  • body element contains all visible content
  • HLTM attributes are used at beginning of element to add details on code
  • h1-h6 is for different levels of headings. p element is for paragraphs. Ul and Ol are for lists. Br element creates line break

CSS

  • CSS Stylesheets determines how the content of a webpage is displayed for users. Css assigns style rules to different elements of an HTML page.
  • 3 ways to use a CSS stylesheet
    1. inline CSS: A CSS style attribute is added to an individual HTML tag by adding the code inside the HTML file
    2. internal CSS: Use a style element within head element to create a CSS stylesheet within the HTML file, where any code will apply to all of the webpage
    3. external CSS: A seperate file contains the CSS rules we want, link to HTML by adding link element in the head element
  • Declarations=CSS Property + Property Value
  • When applying CSS rules to multiple selectors, use a comma between them
  • class attributes start with a (.) and can apply the same CSS style to multiple elements. Id attributes start with a (#) to add CSS style to one specific element.

Git

  • Ls is used to see list of any directory
  • Branches are different versions of code being edited; should have one for each type of change of being done. Changes will not be added to main until they are merged
  • create a new branch by using:git checkout -b feature/""
  • To push changes to remote branch from feature branch use: git push origin feature/""
  • git add to add fies
  • To save all changes made to a file, use: git add -A
  • to commit your changes, use: git commit -m"your message"
  • We always need upload changes back to the main branch to keep it updated. To pull in git's base branch main after saving changes use: git pull origin main
  • To push changes to remote repo use: git push origin (branch name)
  • Send pull request with a description for approval. Then merge the request to merge changes.
